You identify likely duplicate person names in an academic conference people database.

You are given a JSON array named names, where each element is one person's name.
Your task is to find likely duplicates caused by variants in names, including:
- with vs without middle name/initial
- accents/diacritics vs plain ASCII spelling
- role suffixes/prefixes added to a name (for example, "Joe Smith Area Chair")
- minor spelling variants and common transliteration variants

Return ONLY structured output with a single field:
- groups: array of duplicate groups

Each group must contain:
- primary_name: string, exactly one existing name from the input to keep as canonical
- duplicate_names: array of strings, each exactly one existing name from the input and different from primary_name
- rationale: short string explaining why they likely match

Rules:
1) Be conservative: only include high-confidence likely duplicates.
2) Do not invent names not present in the input.
3) Do not include the same name in multiple groups.
4) Do not include singleton groups.
5) Prefer keeping the cleaner, role-free, more person-like name as primary_name.

Input JSON:
{names}
